caddy是服务器的服务器，它能运行时配置，同时管理很多网站。

_______________________________________________________________________________
caddy run 运行

_______________________________________________________________________________
json配置
localhost:2019是caddy的配置端口，只能本机访问
用curl post把配置文件传上去，就能运行时修改一切配置（比如端口）：
curl localhost:2019/load -H "Content-Type: application/json" -d @caddy.json

_______________________________________________________________________________
Caddyfile配置
json才是正式配置文件，Caddyfile会被转化为json，caddyfile本身就是一个adapter。
caddy adapt命令可以直接把Caddyfile转化为对应json。

caddy run会自动使用当前路径下的Caddyfile，也可以指定路径：
caddy run --config /path/to/Caddyfile
可以是任意格式的配置文件

_______________________________________________________________________________
API配置
json/Caddyfile都是配置文件，配置文件生效，本质都是用API操作的，直接用API也是一个方法。

_______________________________________________________________________________
caddy start后台运行（windows上好像不行）
caddy stop 后台停止

_______________________________________________________________________________
更改配置
可以停止运行，更改后重启
可以运行时把json配置文件怼到2019端口
可以运行时用命令重新加载配置文件：caddy reload --config caddy.json
